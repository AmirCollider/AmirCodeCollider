## 🧩 What's inside

```
.
├── index.html               # whole site: HTML + CSS + JS + the 3-language dictionary
├── functions/
│   └── api/
│       ├── order.js         # POST /api/order      -> store request + post a Telegram card
│       └── telegram.js      # POST /api/telegram   -> admin webhook bot  · GET = health probe
├── assets/
│   └── logo.png             # cute placeholder mark — replace with your own
├── _headers                 # basic security headers
└── README.md
```

## 🛠 Tech & architecture

- **Static front-end** — one `index.html`, no framework, no build step. Three languages (FA / EN / JA), dark & light themes, both remembered per visitor.
- **Cloudflare Pages Functions** — `/api/order` validates the form (honeypot + server-side checks), silently drops blocked senders, stores the request, and pushes a tidy management card to Telegram.
- **Telegram admin bot** — `/api/telegram` is a private webhook bot that self-creates its schema, self-registers its slash commands, and lets you browse, paginate, change status, delete, and block/unblock — all from inline buttons.
- **Cloudflare D1** — a serverless SQLite database (bound as `DB`) that persists every request and the blocklist.

## 🚀 Deploy (Cloudflare Pages)

1. Push this folder to **`AmirCollider/AmirCodeCollider`**:
   ```bash
   git init
   git add .
   git commit -m "Launch amircodecollider"
   git branch -M main
   git remote add origin https://github.com/AmirCollider/AmirCodeCollider.git
   git push -u origin main
   ```
2. Cloudflare → **Workers & Pages** → create a Pages project from the repo.
   - Framework preset: **None**
   - Build command: *(empty)*
   - Build output directory: `/`
3. Add the environment variables and the **D1 binding** (see below), then **redeploy** so they load.
4. Point Telegram at the webhook (run once):
   ```bash
   curl "https://api.telegram.org/bot<TELEGRAM_BOT_TOKEN>/setWebhook?url=https://amircodecollider.pages.dev/api/telegram&secret_token=<TELEGRAM_WEBHOOK_SECRET>"
   ```
5. Open the site, submit the form once — the request should arrive in your Telegram.

## 🔐 Environment & bindings

**Settings → Variables and secrets:**

| Name | Required | Purpose |
|------|:--------:|---------|
| `TELEGRAM_BOT_TOKEN` | ✅ | Bot token from BotFather |
| `AmirCollider` *(or `TELEGRAM_CHAT_ID`)* | ✅ | Your numeric Telegram chat id (admin). `TELEGRAM_CHAT_ID` wins if both are set |
| `TELEGRAM_WEBHOOK_SECRET` | ⭕ | Secret echoed in the webhook header so only Telegram can call `/api/telegram` |

**Settings → Bindings:** add a **D1 database** bound as **`DB`** — it powers storage, status tracking, and block/unblock. *(Without it, `/api/order` still delivers a simple card to Telegram.)*

## 🧪 Run locally (optional)

```bash
npm i -g wrangler
```
